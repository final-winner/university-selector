# 智能结论生成后端规则伪代码

## 核心逻辑说明
本模块负责根据两所院校的对比数据，自动生成3行以内的智能对比结论，结合复试线、难度分、学科评估等关键指标，提供简洁有指导性的建议。

## 伪代码实现

```java
// 智能结论生成服务
class IntelligentConclusionService {
    
    // 生成对比结论
    public String generateComparisonConclusion(University uni1, University uni2) {
        // 参数校验
        if (uni1 == null || uni2 == null) {
            return "无法生成对比结论：院校数据不完整";
        }
        
        // 提取关键指标
        int aScore = uni1.getLatestRetestScore();
        int bScore = uni2.getLatestRetestScore();
        String aTrend = uni1.getRetestScoreTrend();
        String bTrend = uni2.getRetestScoreTrend();
        String aLevel = uni1.getLevel();
        String bLevel = uni2.getLevel();
        String aProvince = uni1.getProvince();
        String bProvince = uni2.getProvince();
        String aRating = uni1.getSubjectRating();
        String bRating = uni2.getSubjectRating();
        double aDifficulty = uni1.getDifficultyScore();
        double bDifficulty = uni2.getDifficultyScore();
        
        // 规则1：复试线差距大且趋势明显
        if (Math.abs(aScore - bScore) > 20) {
            if (aScore > bScore) {
                if (isPositiveTrend(aTrend) && isNegativeTrend(bTrend)) {
                    return String.format("%s复试线更高且呈上升趋势，竞争更激烈；%s难度相对适中，可优先考虑。", 
                                        uni1.getName(), uni2.getName());
                }
            } else {
                if (isPositiveTrend(bTrend) && isNegativeTrend(aTrend)) {
                    return String.format("%s复试线更高且呈上升趋势，竞争更激烈；%s难度相对适中，可优先考虑。", 
                                        uni2.getName(), uni1.getName());
                }
            }
        }
        
        // 规则2：同层次同地区
        if (aLevel.equals(bLevel) && aProvince.equals(bProvince)) {
            // 进一步比较学科评估
            if (compareSubjectRating(aRating, bRating) > 0) {
                return String.format("%s和%s同属%s院校且位于同一省份，%s学科评估等级更高，学术实力更强。", 
                                    uni1.getName(), uni2.getName(), aLevel, uni1.getName());
            } else if (compareSubjectRating(aRating, bRating) < 0) {
                return String.format("%s和%s同属%s院校且位于同一省份，%s学科评估等级更高，学术实力更强。", 
                                    uni1.getName(), uni2.getName(), aLevel, uni2.getName());
            } else {
                return String.format("%s和%s同属%s院校且位于同一省份，可结合学科适配性选择。", 
                                    uni1.getName(), uni2.getName(), aLevel);
            }
        }
        
        // 规则3：学科评估差距明显
        if (compareSubjectRating(aRating, bRating) > 1) {
            return String.format("%s学科评估等级%s高于%s的%s，学术实力更强，建议优先选择。", 
                                uni1.getName(), aRating, uni2.getName(), bRating);
        } else if (compareSubjectRating(aRating, bRating) < -1) {
            return String.format("%s学科评估等级%s高于%s的%s，学术实力更强，建议优先选择。", 
                                uni2.getName(), bRating, uni1.getName(), aRating);
        }
        
        // 规则4：难度评分差距明显
        if (Math.abs(aDifficulty - bDifficulty) > 2.0) {
            if (aDifficulty > bDifficulty) {
                return String.format("%s难度评分%.1f高于%s的%.1f，%s竞争相对激烈，%s难度适中。", 
                                    uni1.getName(), aDifficulty, uni2.getName(), bDifficulty, 
                                    uni1.getName(), uni2.getName());
            } else {
                return String.format("%s难度评分%.1f高于%s的%.1f，%s竞争相对激烈，%s难度适中。", 
                                    uni2.getName(), bDifficulty, uni1.getName(), aDifficulty, 
                                    uni2.getName(), uni1.getName());
            }
        }
        
        // 默认规则：综合建议
        return String.format("%s和%s各有优势，建议结合复试线、招生人数和个人学科兴趣综合判断。", 
                            uni1.getName(), uni2.getName());
    }
    
    // 辅助方法：判断趋势是否为正
    private boolean isPositiveTrend(String trend) {
        return trend != null && trend.startsWith("+");
    }
    
    // 辅助方法：判断趋势是否为负
    private boolean isNegativeTrend(String trend) {
        return trend != null && trend.startsWith("-");
    }
    
    // 辅助方法：比较学科评估等级
    // 返回值：1(前者更高), -1(后者更高), 0(相同), -999(无法比较)
    private int compareSubjectRating(String rating1, String rating2) {
        if (rating1 == null || rating2 == null) return -999;
        if (rating1.equals(rating2)) return 0;
        
        // 学科评估等级顺序：A+ > A > A- > B+ > B > B- > C+ > C > C-
        String[] ranks = {"A+", "A", "A-", "B+", "B", "B-", "C+", "C", "C-"};
        int index1 = -1, index2 = -1;
        
        for (int i = 0; i < ranks.length; i++) {
            if (ranks[i].equals(rating1)) index1 = i;
            if (ranks[i].equals(rating2)) index2 = i;
            if (index1 != -1 && index2 != -1) break;
        }
        
        if (index1 == -1 || index2 == -1) return -999;
        return index1 < index2 ? 1 : -1;
    }
}

// 控制器层示例
@RestController
@RequestMapping("/api/comparison")
class ComparisonController {
    
    @Autowired
    private IntelligentConclusionService conclusionService;
    
    @Autowired
    private UniversityService universityService;
    
    @PostMapping("/conclusion")
    public ResponseEntity<Map<String, String>> getConclusion(@RequestBody ComparisonRequest request) {
        try {
            University uni1 = universityService.getUniversityById(request.getUniversityId1());
            University uni2 = universityService.getUniversityById(request.getUniversityId2());
            
            String conclusion = conclusionService.generateComparisonConclusion(uni1, uni2);
            
            Map<String, String> response = new HashMap<>();
            response.put("conclusion", conclusion);
            return ResponseEntity.ok(response);
        } catch (Exception e) {
            // 日志记录
            return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR)
                                .body(Map.of("conclusion", "生成结论失败，请稍后重试"));
        }
    }
}
```

## 关键规则说明

1. **复试线差距规则**：当两所院校复试线差距大于20分且趋势相反时，优先生成基于竞争激烈程度的建议

2. **同层次同地区规则**：当两所院校属于同一层次（如985/211）且位于同一省份时，重点比较学科评估等级

3. **学科评估差距规则**：当学科评估等级差距明显时，基于学术实力给出建议

4. **难度评分差距规则**：当难度评分差距大于2.0时，基于竞争难度给出建议

5. **默认综合规则**：当以上规则都不适用时，给出综合建议

## 集成建议

1. **接口设计**：提供RESTful接口，接收两个院校ID，返回智能结论

2. **缓存策略**：对频繁请求的对比结论进行缓存，提高响应速度

3. **扩展性**：规则引擎可设计为可配置的，便于后续添加或修改规则

4. **国际化**：支持多语言结论生成，适应不同用户需求

5. **测试覆盖**：针对各种规则组合编写单元测试，确保结论生成的准确性
